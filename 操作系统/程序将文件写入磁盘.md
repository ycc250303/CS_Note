## Java 程序层（应用层）

假设代码为：

```java
FileOutputStream fos = new FileOutputStream("test.txt");
fos.write("Hello World".getBytes());
fos.close();
```

* 创建 FileOutputStream
  * Java 调用 **FileOutputStream 构造函数**
  * JVM 调用 **本地方法（native method）**
  * 进入操作系统 **open() 系统调用**
    * 查找当前进程的 **文件描述符表**
    * 在文件系统中找到 `test.txt`
    * 如果不存在则创建
    * 返回一个 **文件描述符 fd**
  * JVM 将这个 fd 存在 `FileDescriptor` 对象中
  * ```
    FileOutputStream
         │
         └── FileDescriptor(fd=3)
    ```

## Java IO 写入

```java
fos.write("Hello World".getBytes());
```

* 将 String 转为 byte[] 类型，转为 ASCII 字节码
* 调用 `FileOutputStream.write()` ，进入 `writeBytes()` 本地方法

## JVM 层

* JVM 调用 **C语言实现的 native 方法**
* 调用操作系统 `write(fd, buffer, length)`
  * fd = 3
  * buffer = Hello World 字节数组
  * length = 11
* 进入 OS 内核态

## 操作系统层

* 系统调用 write 函数，CPU 从用户态进入内核态
* 查找文件对象
  * 操作系统会通过 fd 找到 inode 文件元数据，其中包含
    * 文件大小
    * 磁盘块地址
    * 权限
    * 时间
* 数据写入页缓存 Page Cache

## 文件系统层

* 分配磁盘块
  * 文件系统找到空闲磁盘块，将 Page Cache 缓存放到空闲 block
* 更新 inode
  * 更新文件大小，block 位置，修改时间

## 延迟写回磁盘

* Linux 采用延迟写，在以下时间写入磁盘
  * 内核定时刷盘
  * Page Cache 太大
  * Java 调用 fsync

## 块设备层

* 内核决定写入磁盘，Linux 操作
  * 合并写请求
  * 排序 IO 请求

## 磁盘驱动层

* 负责构造磁盘指令

## 磁盘硬件层

### 机械硬盘

* 磁头寻道
* 找到磁道
* 写入磁盘

### SDD

* 页写入
* 块擦除
* 垃圾回收
* 磨损均衡

## close 过程

```java
FileOutputStream.close()
      ↓
native close()
      ↓
close(fd)
```

* 刷新文件状态
* 释放 fd
* 更新 inode
